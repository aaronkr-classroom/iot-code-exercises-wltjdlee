1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
센서는 온도 , 습도 빛등의  물리적 세계의 데이터를 감지하여 전기적 신호로 변환하는 입력 장치이며 시스템이 외부 환경을 인지하게 하는 감각 기관의 역할을 합니다.
액추에이터는 제어부로부터 받은 전기적 신호를 바탕으로 회전, 발열등 물리적인 동작을 수행하는 출력 장치입니다. 시스템이 외부 환경에 물리적 변화를 가하는 근육의 역할을 합니다.

  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
디바이스와 대시보드에서 동일한 언어를 사용하지 않는 이유 는 자원 제약성과 개발목적의 차이 떄문입니다. 아두이노 ESP32 등과 같은 임베디드 디바이스는 메모리와 연산 능력이 제한적입니다. 따라서 하드웨어 직접 제어가 가능하고 효율성이 높은 C 나 C++ 같은 저수준 언어를 주로 사용게 됩니다 또한 개발 목적의 차이로 대시보드는 사용자 UI 와 UX 제공과 대규모 데이터 처리가 목적입니다. 따라서 웹 인터페이스 구현이 용이하고 생산성이 높은 JavaScript. Python 등의 고수준 언어를 사용하는 것이 효율적입니다.
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
비동기 통신을 통해 송신자와 수신자가 직접 연결될 필요가 없어 시스템 유연성이 확보되며
다수의 디바이스가 브로커 하나를 통해 효율적으로 통신할 수 있어 네트워크 부하가 관리되거나 디바이스가 일시적으로 오프라인 상태가 되어도 브로커가 메시지를 유지하거나 상태를 관리할 수 있습니다.

2. Q2
  - (b) Mosquitto와 Node-RED에 대해 설명하고 , 실행에 적합한 컴퓨팅 환경을 제시하십시오.
IoT 네트워크 시스템의 핵심 소프트웨어 구성 요소인 Mosquitto와 Node-RED는 상호 보완적인 역할을 수행하며 효율적인 데이터 흐름을 생성합니다. 우선 Mosquitto는 MQTT 프로토콜을 기반으로 하는 오픈소스 메시지 브로커로서, 매우 경량화된 설계를 통해 시스템 자원 소모를 최소화하면서도 메시지의 발행 및 구독 및 구조를 안정적으로 관리하는 핵심 허브 역할을 담당합니다. 이와 유기적으로 연동되는 Node-RED는 플로우 기반의 시각적 프로그래밍 도구로, 복잡한 텍스트 코딩 대신 직관적인 노드 연결 방식을 통해 하드웨어 제어 로직을 설계하고 다양한 외부 API 서비스와의 연동을 용이하게 합니다. 이러한 두 도구는 모두 저사양 환경에서도 높은 퍼포먼스를 유지하도록 최적화되어 있으므로, 라즈베리 파이와 같은 싱글 보드 컴퓨터나 리눅스 기반의 미니 PC, 혹은 확장성이 뛰어난 클라우드 환경의 Ubuntu 인스턴스 등을 컴퓨팅 환경으로 구축하여 운용하는 것이 공학적으로 가장 권장됩니다.

3. 3개 이상의 개발 보드를 찾아서 특징과 장단점을 비교하세요.
1. Arduino Uno(아두이노)
Arduino Uno는 8비트 마이크로컨트롤러(MCU)를 기반으로 하는 가장 표준적인 개발 보드입니다. 하드웨어 구조가 직관적이고 내구성이 뛰어나 입문자가 센서와 액추에이터의 동작 원리를 익히기에 가장 적합합니다. 다만, 자체적인 통신 기능이 내장되어 있지 않아 별도의 실드가 필요하며, 연산 성능과 메모리가 제한적이어서 복잡한 데이터 처리나 대규모 IoT 시스템 구현에는 한계가 있습니다.

2. ESP32
ESP32는 저렴한 가격에도 불구하고 와이파이와 블루투스 기능을 내장한 시스템 온 칩 형태의 보드입니다. 저전력 설계가 되어 있어 배터리로 구동되는 IoT 디바이스 제작에 매우 유리하며, Arduino Uno보다 월등히 높은 처리 속도를 자랑합니다. 다만, 보드 크기가 작아 가용 가능한 핀수가 상대적으로 적고, 초기 개발 환경을 설정하는 과정이 입문자에게는 다소 복잡하게 느껴질 수 있다는 단점이 있습니다.

3. Raspberry Pi 4
Raspberry Pi 4는 앞선 보드들과 달리 리눅스 운영체제를 구동할 수 있는 싱글 보드 컴퓨터입니다. ARM 프로세서를 탑재하여 고성능 연산과 멀티태스킹이 가능하며, Python이나 Java와 같은 고수준 언어를 자유롭게 활용할 수 있다는 점이 가장 큰 강점입니다. 따라서 단순한 센서 제어를 넘어 영상 처리나 데이터베이스 서버 운영 등에 적합합니다. 하지만 다른 보드들에 비해 가격이 높고 소모 전력이 크며, 별도의 OS 관리가 필요하다는 운영상의 부담이 존재합니다.

4. 스마트 홈 구축을 위한 아이디어를 다음 조건을 준수하여 정리해보세요. 
  - 전원 제어, 보안, 환경 제어 포함
  - 사용할 센서 목록 작성
  - 대시보드 구성 항목 정리
  - 시스템 통제 시나리오 작성

제가 구축한 시스템은 1인 가구가 겪는 현실적인 문제인 보안 취약성, 외출 시 가전제품 관리의 부주의, 그리고 환기가 어려운 주거 환경을 개선하기 위해 설계된 통합 솔루션입니다. 
시스템의 주요 목적은 거주자의 개입을 최소화하면서도 에너지 효율, 보안, 환경 제어를 동시에 달성하는 데 있습니다.

시스템 구축을 위한 하드웨어 구성으로는, 실내 거주 여부를 파악하 인체 감지 센서, 현관문과 창문의 상태를 확인하는 자기식 도어 센서, 외부 침입 시도를 감지하는 진동 센서, 실내 쾌적도를 측정하는 온습도 센서 , 그리고 공기질 모니터링을 위한 미세먼지 센서를 데이터 수집 장치로 활용합니다. 
이에 대응하는 물리적 제어 장치로는 가전제품의 전원을 관리하는 2채널 릴레이 모듈, 창문 개폐 및 보조 잠금장치 구동을 위한 서보 모터, 위급 상황을 알리는 장치, 그리고 시스템 상태를 색상으로 피드백하는 RGB LED가 포함됩니다.

사용자는 Node-RED 기반의 통합 대시보드를 통해 집안의 모든 상태를 실시간으로 관리합니다. 대시보드에는 가전별 전력 사용량을 시각화하는 에너지 그래프, 실내외 공기질 수치에 따른 환기 실시 가 배치됩니다. 또한, 비상 상황 시 경보를 울리는 위험 버튼과 원격 창문 제어 슬라이더를 통해 언제 어디서든 집안 환경을 통제할 수 있습니다.
이 시스템의 운영 시나리오는 다음과 같은 지능형 로직을 따릅니다. 
첫째, 외출 및 에너지 세이빙'시나리오에서는 도어 센서가 닫히고 일정 시간 내부 움직임이 감지되지 않으면 시스템이 자동으로 외출 모드로 전환되어 불필요한 가전 전원을 릴레이로 차단합니다. 
둘째, 복합 방범 시나리오에서는 취침 시간이나 외출 중 문에 일정 수준 이상의 진동이 감지되거나 비정상적인 개폐가 확인될 경우, 즉시 경보를 울리고 실내 LED를 최대 밝기로 점등하여 침입자에게 경고하며 사용자에게 긴급 알림을 전송합니다. 
셋째, 자동 환경 최적화 시나리오에서는 미세먼지 수치가 기준치를 초과할 경우 실외 대기질 데이터를 확인하여, 외부 공기가 깨끗하다면 서보 모터를 구동해 창문을 자동으로 열어 환기하고, 습도가 지나치게 높을 때는 사용자에게 제습기 가동 알림을 보내 곰팡이 발생을 사전에 방지하고자 합니다.
